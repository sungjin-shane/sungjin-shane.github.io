Scope in programming

Scope

All variables have the scope, so when we use the scope we can handle values in variables appropriately.
Unless the codes are scoped by particular rules, it is very difficult to use the variables. It means coders have to manage all variables. Can you imagine you should control varialbles in long codes, and you should not forget to initialize all the variables.
However, the scope make variables life cycle auctomatically, so coders do not have to bother managing all variables. Coders can easily handle all variables to apply suitable scope rule on their codes.

Baisically, there are two types of scope in javascript.

Global Scope
The global scope is accessible anywhere in your JavaScript code. If we begin writing code in JavaScript, we are already in the global scope. When you want to make varialbes which has whole life cycle in our source codes You can use the Glocal scope.

Local Scope
Local scope is more complex. Locally scoped variables are only visible and accessible within their local scopes where they are defined. We can think of local scope as any new scope that you create within the global scope. Codes are unable to access the values of variables if they try to use the variables in our of the scope. Simply, these locally scoped variables can only be accessed within the function that they are defined.

let className = evt.target.className; //Local variables which can use within fuction evtClick();
